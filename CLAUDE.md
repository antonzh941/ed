# CLAUDE.md — AI-платформа подготовки к ОГЭ

## О проекте

Сервис подготовки к ОГЭ по 4 предметам (русский язык, математика, география, история).
Ключевая фича — сократический ИИ-наставник: не даёт готовых ответов, ведёт ученика
к решению наводящими вопросами. Аудитория — школьники РФ (несовершеннолетние).

## Стек (не менять без обсуждения)

- Next.js (App Router) + TypeScript, монорепо
- PostgreSQL + Prisma; векторный поиск — **pgvector** (НЕ Qdrant, НЕ отдельная векторная БД)
- Redis + BullMQ — фоновые задачи (генерация заданий, аналитика)
- Vercel AI SDK — стриминг, structured output
- Langfuse (self-hosted) — трейсинг всех LLM-вызовов
- Авторизация: VK ID. Оплата: ЮKassa

## AI-слой: архитектурные правила

### Gateway
- Все LLM-вызовы — ТОЛЬКО через единый gateway-модуль (`lib/ai/gateway`).
  Никаких прямых вызовов провайдеров из бизнес-логики.
- Контракт — OpenAI-совместимый API. Провайдер и модель задаются конфигом по роли.
- Основной провайдер: DeepSeek (V4 Flash — диалог, V4 Pro — генерация/проверка).
  Fallback-провайдер обязателен, переключение при недоступности.
- Псевдонимизация: в LLM уходит только учебный контент. Никаких имён, email,
  user ID, телефонов (152-ФЗ, данные несовершеннолетних).

### Четыре роли (отдельные пайплайны, отдельные промпты)
1. **Tutor** — сократический диалог
2. **Generator** — генерация заданий (только офлайн через BullMQ, см. ниже)
3. **Grader** — проверка решений
4. **Explainer** — объяснение теории (с RAG по pgvector)

Промпты хранятся в prompt registry (БД или `prompts/` в git), версионируются.
Хардкод промптов в коде запрещён.

### Сократический режим (критично)
- Сессия решения — конечный автомат: understanding → plan → steps → check → reflect.
  Текущая фаза хранится на сервере, подставляется в промпт.
- Лестница подсказок, 3 уровня: наводящий вопрос → указание метода → разбор первого шага.
  Полный ответ тьютор НЕ выдаёт. После N неудач — переход в отдельный режим «разбор».
- Эталонный ответ хранится в БД, тьютор получает его в системном промпте
  с жёсткой инструкцией + few-shot отказов на провокации.
- Выходной фильтр: каждая реплика тьютора проверяется на утечку эталона
  (регулярка для кратких ответов, дешёвая модель для развёрнутых) до отправки ученику.
- Попытки джейлбрейка логируются в Langfuse с тегом `jailbreak-attempt`.

### Генерация заданий — ТОЛЬКО офлайн-конвейер
Никогда не генерировать задания в рантайме по запросу пользователя.
Пайплайн (BullMQ): генерация по образцу → автовалидация → банк → выдача из банка.
- Математика: эталон пересчитывается кодом (mathjs); параметризуемые задания —
  через шаблоны с подстановкой чисел.
- Гуманитарные: вторая модель (V4 Pro) решает задание вслепую;
  несовпадение с эталоном генератора = брак.
- Структура задания — structured output (JSON: условие, варианты, эталон,
  допустимые варианты записи, тема кодификатора, объяснение).

### Проверка ответов
- Краткий ответ → детерминированное сравнение с эталоном в БД
  (учитывать допустимые варианты: «трактам <или> тракт»). Без LLM.
- Развёрнутый ответ → Grader, structured output строго по критериям ФИПИ
  (каждый критерий — отдельное поле с баллом и обоснованием-цитатой).
- Оценка сочинений всегда сопровождается дисклеймером «ориентировочная».

### Аналитика прогресса
- Метрики считает SQL (materialized views), НЕ LLM.
- LLM используется только для еженедельного текстового пересказа
  уже посчитанных цифр (фоновый джоб).

## Данные

- Банк заданий: реляционные таблицы (предмет, номер задания, тема кодификатора,
  условие, эталон, критерии). Выборка по фильтрам, не векторным поиском.
- Теория: фрагменты по темам кодификатора + эмбеддинги в pgvector
  (RAG только для Explainer и подмешивания теории тьютору).
- Исходники для наполнения: оцифрованные демоверсии ФИПИ 2025
  (русский, математика, география, история) — в `data/rag/`.

## Конвенции

- Язык кода и комментариев — английский; пользовательские тексты и промпты — русский.
- Структурированные ответы моделей — через zod-схемы + AI SDK `generateObject`.
- Каждый LLM-вызов обязан проходить через Langfuse-трейсинг (роль, модель,
  токены, стоимость, sessionId).
- Тесты: golden set диалогов для регрессии промптов (прогон при смене модели/промпта).
- Миграции — только через Prisma Migrate.

## Чего НЕ делать

- Не добавлять Qdrant/Weaviate/Pinecone — pgvector достаточно.
- Не вызывать LLM для подсчёта статистики.
- Не генерировать задания синхронно в запросе пользователя.
- Не отправлять PII во внешние API.
- Не хардкодить названия моделей в бизнес-логике — только конфиг gateway.
- Не давать тьютору выдавать готовый ответ ни при каких формулировках ученика.
